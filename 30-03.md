Built the HeatMap component — a 52-week grid going backwards from today, each cell colored based on commit count. Added tooltips showing the date and commit count on hover, and made it horizontally scrollable on small screens.



A contribution heatmap works by mapping dates to activity counts and rendering them as a grid. The data comes from GitHub's public events API which returns push events with timestamps. Each cell represents one day, and the color intensity is determined by how many commits were pushed that day. The grid is built by calculating 52 weeks backwards from today's date, grouping events by their ISO date string, and assigning a color from a predefined scale — empty days get the darkest background, and higher commit counts get progressively brighter shades of the accent color.The scrollable design is intentional — on smaller screens a 52-week grid would overflow and break the layout if it tried to fit everything in one line. Making it horizontally scrollable keeps the data intact without compressing the cells to the point where they become unreadable. This is the same approach GitHub uses on mobile.
