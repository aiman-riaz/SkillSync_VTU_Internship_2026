Finished compare mode — a second GitHub username can be entered at the bottom of any report page. The app fetches that user's full data and overlays their scores on the radar chart as a second dataset. A winner table below highlights the higher score per category in green.



Chart.js supports multiple datasets on the same radar chart natively. Each dataset gets its own border color, fill color and point color, and both are rendered simultaneously on the same axes. The fill opacity is kept low so neither dataset obscures the other — you can see both polygons clearly even when they overlap heavily. This is more informative than two separate charts because it makes the relative strengths and weaknesses of each developer immediately visible at a glance rather than requiring the user to mentally compare two separate images.



The winner table breaks the comparison down category by category because the overall score alone can be misleading. A developer with a very high activity score can have a similar overall score to one with very high community impact, but they represent completely different profiles. Showing which user wins each category gives a more honest and detailed picture of where each developer's strengths actually lie, which is more useful for someone trying to make a hiring decision.
