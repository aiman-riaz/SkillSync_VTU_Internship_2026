Built the RepoList component showing the top 6 repositories sorted by star count. Each card displays the repo name, description, language pill with its associated color, star count and fork count, and links out to the actual GitHub repository page.



Language pills use a predefined color map where each programming language has a fixed hex color — JavaScript gets yellow, Python gets blue, Rust gets orange and so on. These colors are the same ones GitHub uses in its own language breakdown bar, which makes the UI immediately familiar to any developer looking at the report. The colors are applied both to the dot inside the pill and to the pill's border, creating a subtle but consistent visual identity per language.



Sorting repos by star count rather than last updated date is a deliberate product decision. Stars are a community signal of quality and usefulness, so showing the highest-starred repos gives recruiters and hiring managers the most impressive work first. Forks are shown alongside stars because a high fork count often indicates that other developers are actively building on top of the project, which is another strong signal of impact.
