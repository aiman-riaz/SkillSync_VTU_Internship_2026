How it Works (HTTP Methods)
REST APIs map standard HTTP verbs to CRUD (Create, Read, Update, Delete) operations on resources: 
GET: Retrieves a resource (e.g., GET /users/123).
POST: Creates a new resource (e.g., POST /users).
PUT: Updates or replaces an entire existing resource.
PATCH: Partially updates a resource (only modified fields).
DELETE: Removes a specific resource. 
Key Components
Resources & Endpoints: Data (like a "user" or "product") is treated as a resource identified by a unique URI/URL (e.g., https://api.example.com).
Data Formats: Information is typically exchanged in JSON (the industry standard) or XML.
Status Codes: Servers return three-digit codes to indicate the outcome: 200 OK (success), 201 Created, 400 Bad Request, 404 Not Found, or 500 Internal Server Error.
Authentication: Secure access is managed via OAuth 2.0, API Keys, or Bearer Tokens.
