Fixed mobile layout breaking on small screens, a scoring crash on accounts with no public events, a ScoreCard crash when the GitHub name field is null, heatmap month labels being off by one week, and the compare endpoint failing silently when one username doesn't exist.




Edge cases like null fields and empty arrays are extremely common when consuming third-party APIs. GitHub does not guarantee that every field on a user object will be populated — new accounts often have no bio, no name, no website and no public events at all. Without defensive checks these missing values propagate through the application and eventually cause runtime errors that crash the UI. The fixes involved adding null coalescing operators and empty array fallbacks at the points where the data is first consumed, which is the correct place to handle it rather than deeper in the rendering logic.



The heatmap month label offset was a pure date math bug. The grid starts from the Sunday of the week 52 weeks ago, but the month label logic was reading the first day of each week without accounting for that Sunday alignment. This caused labels to appear one cell too early, showing the wrong month name above certain columns. Fixing it required recalculating the label position based on when the first day of a new month actually falls within the grid rather than assuming it aligns with a week boundary.
