Built the GitHub service using Octokit — getUser, getRepos, getEvents and getRepoContents all working. Added proper error handling so missing usernames return a 404 and failed content fetches return an empty array instead of crashing.


The Wrapper Pattern
Instead of your app making raw HTTP requests everywhere, you wrapped Octokit into specific functions. This made your code decoupled. If GitHub ever changes their API version or if you want to swap Octokit for another library later, you only have to change that one service file instead of hunting through your entire project.


Error Normalization
You practiced what’s called Error Normalization. External APIs are unpredictable—they can timeout, rate-limit you, or just go down.
By catching a missing username and turning it into a 404, you gave your frontend a clear, actionable signal.
By returning an empty array for failed content fetches, you followed a "fail-safe" design. It prevented the whole page from breaking just because one small part of the data (like a file list) didn't load.
