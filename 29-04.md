Focused entirely on the appointment system which is the core feature of the project. Wrote the booking function with a conflict check that queries the database for an existing non-cancelled appointment on the same doctor, date, and slot before allowing a new one. Added endpoints for patients to view their own appointments and cancel them, and for doctors to view their patient list. The getBookedSlots endpoint is a lightweight public route that the frontend will call when a date is selected to know which slots to disable. Added validation so patients can't cancel completed or already-cancelled appointments.



Business logic validation should happen on the server side even if the client also validates, because API requests can be made directly without going through the UI. An idempotency concern arises with bookings — the same slot should not be bookable twice, which is why both a database-level unique index and a pre-insert query check are used together. HTTP status codes communicate the result of a request: 201 for created, 400 for bad request, 401 for unauthorized, 404 for not found. Mongoose's findOne with a filter is used to check for existing records before inserting. Partial updates with findByIdAndUpdate use the new: true option to return the updated document rather than the old one. Separating patient and doctor appointment routes keeps role permissions explicit.
