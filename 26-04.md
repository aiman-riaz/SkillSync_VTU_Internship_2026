Created all three Mongoose models today which define the shape of every document stored in the database. The User model has a pre-save hook that hashes the password automatically using bcryptjs before saving, so plain text passwords never hit the database. The Doctor model is a separate collection with a reference to User and includes a nested availability array where each entry holds a day name, start time, end time, and a boolean. The Appointment model has a compound unique index on doctor, date, and time slot together which prevents double bookings at the database level.



Mongoose schemas define the structure, data types, and validation rules for documents in a collection. A pre-save hook is middleware that runs automatically before a document is saved, commonly used for password hashing. bcryptjs implements the bcrypt algorithm which is a one-way hashing function.



References between collections (like Doctor.user pointing to a User _id) create a relational-style link in a non-relational database. Schema design at this stage prevents data integrity issues later.
