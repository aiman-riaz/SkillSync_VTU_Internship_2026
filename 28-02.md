### Library vs Framework

**Library**

* Provides functions you call.
* You control the flow.

**Framework**

* Provides structure and controls the flow.
* You fill in the logic.

---

### Why React is Called React

* React updates UI efficiently without full page reload.
* Uses **Virtual DOM** to “react” to changes quickly.

---

### React vs ReactDOM

* React → UI logic and components.
* ReactDOM → Rendering UI in the browser.

---

### Real DOM vs Virtual DOM

**Real DOM**

* Actual webpage structure.
* Full updates are costly.

**Virtual DOM**

* Lightweight copy in memory.
* React compares changes (diffing) and updates only necessary parts.
* Process called **Reconciliation**.
