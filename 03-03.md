Git is a free, open-source version control system (VCS), while GitHub is a web-based platform that hosts Git repositories and provides collaboration features. Essentially, Git is the underlying tool, and GitHub is a service that builds upon Git's functionality to facilitate team collaboration and project management online.

Git is a software that you install locally on your computer. Its core purpose is to track and manage changes in source code and other files efficiently. Key aspects of Git include: 
Distributed Architecture: Every developer has a complete copy of the project's history on their local machine, allowing for offline work and data redundancy.
Core Functions: It allows users to make "commits" (save points or snapshots of the code at a certain time), manage different "branches" for experimenting with new features safely, and "merge" changes back into the main codebase.
Command-Line Interface: Git is primarily used through the command-line, although graphical user interfaces (GUIs) like GitHub Desktop also exist.
Open-Source: It is open-source software, originally developed by Linus Torvalds in 2005.
