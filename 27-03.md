Built the Axios API helper and the SearchBar component. Styled it with a pill shape and accent border on focus, added inline validation for empty input.


Architectural Theory: The Service Layer (Axios Helper)
By centralizing API calls into a helper file, you implement a Service Layer pattern that promotes Separation of Concerns: 
Maintainability: If the API base URL or authentication method changes, you update a single file rather than every individual component.
Boilerplate Reduction: Axios automatically handles JSON transformation (response.data) and treats non-2xx status codes as errors, removing the need for manual .json() parsing or status checks required by the native Fetch API.
Global Logic: Helpers allow you to easily add Interceptors for tasks like adding authorization tokens or logging every request globally.



Design Theory: UX and Visual Affordance
The choice of a pill shape and accent borders is grounded in human-computer interaction (HCI) principles:
Pill Shape & "Friendly" UI: Rounded shapes are often perceived as more approachable and interactive. A pill-shaped search bar has high visual affordance, clearly signalling to the user that it is an input field meant for interaction.
Focus State & Orientation: An accent border on focus provides a critical visual indicator for navigation. It helps users—especially those using keyboards—stay oriented by clearly highlighting which element is currently active.
Color Communication: Using specific colors (like blue for focus or red for errors) leverages standard mental models where color change indicates a change in system state. 



Validation Theory: "Reward Early, Punish Late"
Adding inline validation for empty inputs addresses common friction points in form filling: 
Immediate Feedback: Inline validation speeds up error recovery by alerting users while the task is still fresh in their minds.
Constraint Validation: Validating locally (client-side) before making an API request prevents unnecessary network traffic and reduces server load.
Reducing Abandonment: Research shows nearly 70% of users may abandon a form if they encounter complications. Effective inline validation reduces this friction by preventing the "jarring" experience of submitting a form only to be met with errors.
