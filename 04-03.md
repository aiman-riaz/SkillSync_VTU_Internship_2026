A REST API (Representational State Transfer Application Programming Interface) is a web service that follows a set of architectural principles for communication over the internet, typically using standard HTTP methods. They provide a standardized, lightweight, and scalable way for different software applications (clients and servers) to exchange data.

Resources and URIs: Every piece of data is considered a "resource" and is uniquely identified by a Uniform Resource Identifier (URI), or URL (e.g., /users or /products/123).
HTTP Methods: Standard HTTP methods are used to perform operations on these resources, mapping to common data functions (CRUD: Create, Read, Update, Delete):
GET: Retrieve a resource or collection of resources.
POST: Create a new resource.
PUT: Update an existing resource (replaces the entire resource).
PATCH: Perform a partial update on a resource.
DELETE: Remove a resource.
Representations: The server returns a representation of the resource, typically in JSON format, which is easy for both humans and machines to read. XML is another possible format.
Statelessness: Each request is independent and contains all the information necessary for the server to process it. The server does not store any client session state between requests, which improves scalability.
