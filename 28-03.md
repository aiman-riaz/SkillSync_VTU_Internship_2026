Built the ScoreCard component — avatar, bio, stats, SVG progress ring with score-based color, and animated category bars.


The SVG Progress Ring (Geometry & Math)
Using an SVG instead of a CSS border for the ring allows for precision.
stroke-dasharray & stroke-dashoffset: This is the core "trick." 



Score-Based Color (Conditional Logic): This implements Semantic Color Mapping. By mapping scores (e.g., 0-50 Red, 51-80 Yellow, 81-100 Green), you provide "at-a-glance" cognitive processing so the user doesn't even have to read the number to know if it’s good or bad.



Animated Category Bars (Perceived Performance)
Staggered Animations: If you have multiple categories (e.g., Design, Code, Speed), animating them from 0 to X% on mount creates a sense of "loading life" into the data.
GPU Acceleration: Using transform: scaleX() instead of changing the width property is critical. Changing width triggers a layout reflow (slow); transform only triggers a composite (fast/60fps).



Information Hierarchy (Avatar & Bio)
The "Anchor" Point: The Avatar and Bio serve as the identity anchor. In dashboard theory, the "Who" (Profile) must always be visually linked to the "How Well" (Stats) to provide context to the data.
Micro-copy: The Bio provides the qualitative data that supports the quantitative stats, creating a balanced "Profile Story."



Technical Strategy: Modular Props
To make this component reusable, you likely structured it to accept a user object.
Prop Composition: Instead of passing 10 individual props, passing one data object makes the component easier to implement in lists (like a Leaderboard).
