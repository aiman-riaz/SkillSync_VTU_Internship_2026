Wrote the full auth system on the backend today. The register function handles both patients and doctors, and if the role is doctor it also creates the linked doctor profile in one go. Login compares the submitted password against the stored hash and returns a signed JWT on success. Built the protect middleware that checks the Authorization header, verifies the token, and attaches the decoded user to req.user so any route can access it. Set up the auth routes and ran Postman tests to confirm register, login, and getMe all respond correctly.



Authentication is the process of verifying who a user is. JWT (JSON Web Token) is a compact, self-contained token that encodes user identity and is signed with a secret key so it cannot be tampered with. When a user logs in, the server issues a token which the client stores and sends back in every subsequent request via the Authorization header. Middleware in Express is a function that runs between receiving a request and sending a response, used here to protect routes that require login. bcrypt's compare function checks a plain password against a hash without needing to decrypt it. Role-based access control restricts what actions a user can perform based on their assigned role.
