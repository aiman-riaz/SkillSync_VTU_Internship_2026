About MongoDB aggregation
Essential Pipeline Stages
$match: Filters documents to pass only those that meet specific criteria (similar to a SQL WHERE clause).
$group: Groups documents by a specific key and performs calculations like $sum, $avg, or $count on the group.
$project: Reshapes documents by selecting, removing, or adding new fields to the output.
$sort: Rearranges documents in ascending or descending order.
$lookup: Performs a left outer join to bring in related data from a different collection.
$unwind: Deconstructs an array field, creating a separate document for each element in the array.


Inside these stages, you use Aggregation Expressions. These are functional operators (like $add, $cond, or $filter) that allow you to perform logic within the document.
Conditional Logic: You can use $cond (if-then-else) to create new fields on the fly based on existing data.
Array Handling: Since MongoDB is document-based, aggregation theory focuses heavily on Arrays. The $unwind stage is unique to NoSQL; it "flattens" an array so you can treat each element as a separate document for counting or totaling.
