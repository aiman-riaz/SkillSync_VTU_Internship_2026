Built the Report page with an API call on mount using useEffect, a loading spinner during the fetch, and an error state with a retry button. All components are rendered in a two-column grid. Added a share button that copies the current URL to the clipboard.



useEffect with the username as a dependency means the page automatically re-fetches whenever the URL parameter changes. This is important because React Router keeps the same component instance mounted when navigating between two report URLs — without the dependency the page would show stale data from the previous user. The loading and error states are tracked as separate pieces of state rather than a single status string, which makes the conditional rendering logic cleaner and easier to extend later.



The share button uses the Clipboard API which is available in all modern browsers. When clicked it copies window.location.href to the clipboard and temporarily changes the button text to confirm the action. The URL itself is the shareable report link — because every report lives at a unique route like /report/torvalds, anyone who receives the link lands directly on that user's scorecard without needing to search again. This makes the app genuinely useful for sharing on LinkedIn or sending to a recruiter.
